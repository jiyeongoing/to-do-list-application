# Swipe Todo - Feature Policy

## 1. 목록 정책

| 목록 | 항목의 역할 | 추가 가능 | 완료 가능 | 자동 처리 |
| --- | --- | --- | --- | --- |
| 오늘 | 오늘 실행할 할 일 | O | O | 없음 |
| 데일리 | 매일 반복되는 원본 루틴 | O | X | 매일 오늘에 복사 생성 |
| 계획 | 특정 날짜에 실행할 예약 항목 | O | X | 지정 날짜의 오늘 할 일로 이동 |
| 목적별 리스트 | 특정 날짜의 상황성 체크리스트 | O | 하위 항목만 O | 지정 날짜에 오늘에 카드 노출 |

## 2. 날짜 기준

- 사용자 기기의 현지 시간을 기준으로 동작합니다.
- 하루는 `00:00 ~ 23:59`입니다.
- 앱 실행 시 오늘 생성 여부와 도래한 예약 항목을 확인합니다.
- 오늘 할 일은 날짜값을 함께 저장하며, 오늘 화면에는 현재 날짜 항목만 표시합니다.

## 3. 데일리 자동 생성

| 상황 | 처리 |
| --- | --- |
| 오늘 처음 앱을 실행함 | 활성 데일리 항목을 오늘 목록에 생성 |
| 같은 날 앱을 다시 실행함 | 데일리 항목을 추가로 만들지 않음 |
| 날짜가 00:00을 지나 바뀜 | 활성 데일리 항목을 새 날짜에 미완료 상태로 다시 생성 |
| 데일리 항목을 비활성화함 | 다음 날부터 오늘 목록에 생성하지 않음 |
| 이미 생성된 항목의 원본을 비활성화함 | 오늘 항목은 유지 |
| 새 데일리 항목을 추가함 | 기본 미사용 상태로 생성 |

## 4. 계획 자동 이동

| 상황 | 처리 |
| --- | --- |
| 미래 날짜에 할 일을 작성함 | 선택한 날짜의 계획 목록에 저장 |
| 오늘 날짜에 할 일을 작성함 | 오늘 목록에 바로 저장 |
| 계획에서 오늘 날짜를 선택함 | 오늘 날짜의 할 일을 함께 표시 |
| 지정 날짜에 처음 앱을 실행함 | 해당 날짜의 오늘 할 일로 이동 |
| 지정 날짜에 앱을 열지 않음 | 다음 실행 시 원래 지정 날짜의 할 일로 정리 |
| 이동 후 | 계획 목록에서는 제거 |

예약 항목은 복사가 아니라 이동으로 처리해 중복을 방지합니다.

예시:

```text
5월 30일: "미용실 예약"을 계획에 입력
5월 30일 앱 실행 -> 5월 30일 오늘 목록에 "미용실 예약" 이동
5월 30일 미실행, 5월 31일 실행 -> 5월 30일 계획 기록으로 확인
```

## 5. 목적별 리스트

| 상황 | 처리 |
| --- | --- |
| 계획에서 `+ 리스트`를 누름 | 별도 리스트 편집 페이지로 이동 |
| 편집 페이지에서 `등록`을 누름 | 선택 날짜 화면에 리스트로 등록 |
| 편집 페이지에서 `삭제`를 누름 | 리스트를 제거하고 선택 날짜 화면으로 복귀 |
| 등록된 리스트를 누름 | 해당 날짜에 연결된 상세 체크리스트를 편집 |
| 예: `장보기`, `여행 준비` | 각 리스트는 독립된 상세 화면을 가짐 |
| 지정 날짜가 됨 | 오늘 목록에 리스트 카드로 표시 |
| 리스트 카드를 누름 | 해당 리스트 항목만 보이는 화면으로 이동 |
| 날짜가 지났지만 미완료 항목이 있음 | 완료될 때까지 오늘에 카드 유지 |

목적별 리스트의 하위 항목을 일반 오늘 투두로 풀어서 섞지 않습니다.

## 6. 완료와 삭제

- 완료 처리는 `오늘` 목록에서만 제공합니다.
- 목적별 리스트는 상세 화면에서 하위 항목을 완료 처리합니다.
- 완료 항목은 당일 오늘 화면 하단에 흐리게 표시합니다.
- 날짜가 바뀌면 이전 날짜의 오늘 항목은 오늘 화면에서 제외하고, 계획에서 해당 날짜 기록으로 확인합니다.
- 완료 항목도 기기 내부 데이터에는 저장합니다.
- 삭제 버튼을 누르면 해당 항목은 즉시 삭제합니다.
- MVP에서는 휴지통이나 삭제 복구 기능을 제공하지 않습니다.

## 7. 저장 정책

| 항목 | 정책 |
| --- | --- |
| 저장 위치 | 현재 프로토타입은 사용자의 해당 브라우저 `LocalStorage` |
| 로그인 | 이메일/비밀번호 회원가입 및 로그인 제공 |
| 동기화 | 게스트는 로컬 저장, 로그인 사용자는 계정 저장소로 전환 |
| 오프라인 | 기존 저장 데이터 조회 및 수정 가능 |
| 데이터 유실 가능성 | 브라우저 데이터 삭제 또는 기기 변경 시 복구 불가 |

최초 사용 시 다음 내용을 안내합니다.

> 로그인 없이 이 기기에만 저장됩니다. 브라우저 데이터를 삭제하면 목록을 복구할 수 없습니다.

## 8. 계정 저장 정책

| 상황 | 처리 |
| --- | --- |
| 로그인하지 않음 | 기존처럼 이 기기에 저장 |
| 이메일/비밀번호 로그인 | 계정 저장 흐름으로 전환 |
| Google 계정 연동 | 같은 이메일 계정에 Google ID 연결 |
| Apple 로그인 | iPhone 사용자 편의를 위해 후속 검토 |
| 로그인 직후 | `이 기기 데이터 가져오기` 선택지를 제공 |
| 가져오기 선택 | 로컬 데이터와 서버 데이터를 병합 |
| 가져오기 미선택 | 서버 데이터만 사용하고 로컬 데이터는 보존 |
| 로그아웃 | 서버 저장을 멈추고 게스트 모드로 전환 |

병합 시에는 사용자가 만든 데이터를 임의로 삭제하지 않습니다. 같은 제목의 항목이
있더라도 우선 보존하고, 이후 사용자가 직접 정리할 수 있게 합니다.

현재 정적 PWA에서는 실제 OAuth 서버 연결 전 단계로, Google 로그인 사용자 흐름과
계정 저장 전환 정책을 프로토타입으로 검증합니다.

## 9. 계획 탐색과 순서 변경

- 계획 화면 상단에는 현재 기준 3개월을 빠르게 선택할 수 있는 월 버튼을 둡니다.
- 3개월은 이번 달, 다음 달, 다다음 달을 기준으로 합니다.
- 작은 달력 버튼으로 원하는 날짜를 직접 고를 수 있습니다.
- 선택한 월 안에서는 날짜를 고르고, 좌우 화살표로 이전/다음 날짜로 이동합니다.
- 오늘 할 일, 계획/오늘의 별도 리스트, 데일리 루틴, 리스트 내부 항목은 작은 삼각형 버튼으로 순서를 바꿀 수 있습니다.
- 첫 항목은 아래 이동만, 마지막 항목은 위 이동만 표시합니다.

## 10. 제스처와 접근성

- 오늘 화면에서 오른쪽으로 스와이프하면 `계획`으로 이동합니다.
- 계획 화면에서 왼쪽으로 스와이프하면 `오늘`로 돌아갑니다.
- 스와이프가 어려운 사용자를 위해 `계획하기` 버튼도 제공합니다.
- `계획하기` 버튼으로 진입하면 오늘 날짜를 자동 선택합니다.
- 목적별 리스트 카드는 `열기` 버튼으로 단독 화면에 접근할 수 있습니다.
- 계획 화면에서 리스트를 등록하거나 삭제하면 원래 선택 날짜 화면으로 돌아갑니다.
- 핵심 동작은 제스처만으로 제한하지 않습니다.

## 11. 정책 보류 항목

| 항목 | 보류 이유 |
| --- | --- |
| 오늘 미완료 항목의 다음 날 자동 이월 | 원치 않는 누적이 생길 수 있음 |
| 목적별 리스트 재사용 템플릿 | 먼저 일회성 장보기/여행 목록의 사용성을 검증 |
| 완료 기록 조회 | 핵심 흐름보다 우선순위가 낮음 |
| 이메일/비밀번호 가입 | OAuth 로그인 검증 후 필요하면 추가 |
