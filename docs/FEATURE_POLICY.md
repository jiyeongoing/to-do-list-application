# Swipe Todo - Feature Policy

## 1. 목록 정책

| 목록 | 항목의 역할 | 추가 가능 | 완료 가능 | 이동 |
| --- | --- | --- | --- | --- |
| 오늘 | 오늘 실행할 할 일 | O | O | 없음 |
| 이번 주 | 이번 주 안에 처리할 후보 | O | X | 오늘로 이동 |
| 데일리 | 매일 반복되는 원본 루틴 | O | X | 매일 오늘에 복사 생성 |
| 다음 주 | 다음 주를 미리 준비하는 후보 | O | X | 주 전환 시 이번 주로 이동 |

## 2. 날짜 기준

- 사용자 기기의 현지 시간을 기준으로 동작합니다.
- 하루는 `00:00 ~ 23:59`입니다.
- 한 주는 `월요일 00:00 ~ 일요일 23:59`입니다.
- 앱 실행 시 현재 날짜와 마지막 실행 날짜를 비교해 자동 처리를 수행합니다.

## 3. 데일리 자동 생성

| 상황 | 처리 |
| --- | --- |
| 오늘 처음 앱을 실행함 | 활성화된 데일리 항목을 오늘 목록에 생성 |
| 같은 날 앱을 다시 실행함 | 데일리 항목을 추가로 만들지 않음 |
| 데일리 항목을 비활성화함 | 다음 날부터 오늘 목록에 생성하지 않음 |
| 이미 생성된 오늘 항목의 원본을 비활성화함 | 오늘 항목은 유지 |

예시:

```text
데일리: "물 마시기" 활성화
5월 27일 첫 실행 -> 오늘 목록에 "물 마시기" 1개 생성
5월 27일 재실행 -> 추가 생성 없음
5월 28일 첫 실행 -> 오늘 목록에 새 항목 1개 생성
```

## 4. 주간 이동

| 상황 | 처리 |
| --- | --- |
| 다음 주 목록에 항목을 작성함 | 다음 주 목록에만 표시 |
| 새로운 월요일에 앱을 처음 실행함 | 저장된 다음 주 항목을 이번 주로 모두 이동 |
| 이동 후 | 다음 주 목록은 비워짐 |
| 주가 바뀐 뒤 며칠 후 처음 실행함 | 그 시점에 한 번 이동 |

다음 주 항목은 복사가 아니라 이동으로 처리하여 같은 할 일이 두 목록에
중복으로 남지 않게 합니다.

## 5. 오늘로 이동

| 행동 | 결과 |
| --- | --- |
| 이번 주 항목에서 `오늘로 이동` 선택 | 오늘 목록에 표시되고 이번 주 목록에서는 제거 |
| 이동된 항목 완료 | 오늘 완료 영역에 표시 |
| 이동된 항목 삭제 | 오늘 목록에서만 삭제 |

## 6. 완료와 삭제

- 완료 처리는 `오늘` 목록에서만 제공합니다.
- 완료 항목은 당일 오늘 화면 하단에 흐리게 표시합니다.
- 완료 항목도 기기 내부 데이터에는 저장합니다.
- 삭제 버튼을 누르면 해당 항목은 즉시 삭제합니다.
- MVP에서는 휴지통이나 삭제 복구 기능을 제공하지 않습니다.

## 7. 저장 정책

| 항목 | 정책 |
| --- | --- |
| 저장 위치 | 사용자의 해당 브라우저 내부 저장소 `IndexedDB` |
| 로그인 | 제공하지 않음 |
| 동기화 | 제공하지 않음 |
| 오프라인 | 기존 저장 데이터 조회 및 수정 가능 |
| 데이터 유실 가능성 | 브라우저 데이터 삭제 또는 기기 변경 시 복구 불가 |

최초 사용 시 다음 내용을 안내합니다.

> 로그인 없이 이 기기에만 저장됩니다. 브라우저 데이터를 삭제하면 목록을 복구할 수 없습니다.

## 8. 제스처와 접근성

- `다음 주`는 오른쪽 스와이프로 접근합니다.
- 스와이프가 어려운 사용자를 위해 작은 `다음 주` 버튼도 함께 제공합니다.
- 핵심 동작은 제스처만으로 제한하지 않습니다.

## 9. 정책 보류 항목

아래 항목은 실제 사용 피드백을 받은 뒤 결정합니다.

| 항목 | 보류 이유 |
| --- | --- |
| 오늘 미완료 항목의 다음 날 자동 이월 | 사용자가 원치 않는 누적이 생길 수 있음 |
| 이번 주 미완료 항목의 다음 주 처리 | 자동 이월과 수동 정리 중 선호 확인 필요 |
| 완료 기록 조회 | MVP 핵심 흐름보다 우선순위가 낮음 |
