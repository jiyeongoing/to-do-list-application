## 2. User Stories

| ID | Priority | User Story | Acceptance Criteria |
| --- | --- | --- | --- |
| TD-01 | P0 | 사용자는 오늘 할 일을 추가할 수 있다. | 입력한 항목이 오늘 목록에 즉시 표시되고 재실행 후 유지된다. |
| TD-02 | P0 | 사용자는 오늘 할 일을 완료하거나 삭제할 수 있다. | 완료 상태가 표시되며 삭제된 항목은 목록에서 사라진다. |
| TD-03 | P0 | 사용자는 이번 주 할 일을 따로 등록할 수 있다. | 이번 주 탭에서 등록/삭제할 수 있다. |
| TD-04 | P0 | 사용자는 이번 주 할 일을 오늘로 옮길 수 있다. | 이동 후 오늘 탭에 나타나며 이번 주 목록에서는 제거된다. |
| TD-05 | P0 | 사용자는 데일리 반복 항목을 등록할 수 있다. | 활성 항목이 새 날짜 최초 실행 시 오늘 목록에 1회 생성된다. |
| TD-06 | P0 | 사용자는 스와이프로 다음 주 목록을 열 수 있다. | 기본 화면에서 제스처로 다음 주 화면에 접근하고 돌아올 수 있다. |
| TD-07 | P0 | 다음 주가 되면 예정 항목이 이번 주로 이동한다. | 월요일 최초 실행 시 항목이 중복 없이 이번 주에 표시된다. |
| TD-08 | P0 | 사용자는 로그인 없이 데이터를 유지할 수 있다. | 동일 기기/브라우저에서 앱을 다시 열어도 목록이 유지된다. |
| TD-09 | P0 | 사용자는 홈 화면에서 앱처럼 실행할 수 있다. | PWA 설치 후 standalone 화면으로 열린다. |
| TD-10 | P1 | 사용자는 인터넷 없이 기존 목록을 확인할 수 있다. | 한 번 접속한 뒤 오프라인에서 앱 화면과 저장 데이터를 볼 수 있다. |
| TD-11 | P1 | 사용자는 완료 항목을 구분해 볼 수 있다. | 완료된 오늘 항목이 하단에 분리되어 표시된다. |

## 3. 개발 마일스톤

| 단계 | 범위 | 확인 결과 |
| --- | --- | --- |
| M1. 화면 프로토타입 | 탭, 스와이프 화면, 빈 상태 | 모바일 화면에서 흐름 검토 가능 |
| M2. 기본 투두 | 오늘/이번 주 추가, 완료, 삭제 | 데이터가 기기 내부에 유지됨 |

| M5. 사용자 검증 | 1주 직접 사용 | 다음 개발 우선순위 결정 |
