## 2. User Stories

| ID | Priority | User Story | Acceptance Criteria |
| --- | --- | --- | --- |
| TD-01 | P0 | 사용자는 오늘 할 일을 추가할 수 있다. | 입력 항목이 오늘에 표시되고 재실행 후 유지된다. |
| TD-02 | P0 | 사용자는 오늘 할 일을 완료하거나 삭제할 수 있다. | 완료 상태가 표시되며 삭제 항목은 사라진다. |
| TD-03 | P0 | 사용자는 데일리 반복 항목을 등록할 수 있다. | 활성 항목이 새 날짜 최초 실행 시 오늘에 1회 생성된다. |
| TD-04 | P0 | 사용자는 화면을 넘겨 계획을 열 수 있다. | 제스처 또는 버튼으로 계획 화면에 접근하고 돌아올 수 있다. |
| TD-05 | P0 | 사용자는 선택 날짜에 할 일을 작성할 수 있다. | 항목이 선택한 날짜에 저장되고 재실행 후 유지된다. |
| TD-06 | P0 | 예약 날짜가 되면 할 일이 오늘로 이동한다. | 도래한 항목이 중복 없이 오늘에 표시된다. |
| TD-07 | P0 | 사용자는 날짜에 연결된 목적별 리스트를 만들 수 있다. | 별도 페이지에서 리스트를 등록/삭제하고 선택 날짜 화면으로 돌아갈 수 있다. |
| TD-08 | P0 | 사용자는 오늘 도착한 목적별 리스트만 열어볼 수 있다. | 카드 선택 시 해당 체크리스트만 표시되고 완료 처리할 수 있다. |
| TD-09 | P0 | 사용자는 순서를 조정할 수 있다. | 오늘 할 일, 목적별 리스트, 데일리 루틴, 리스트 항목을 위아래로 이동할 수 있다. |
| TD-10 | P0 | 사용자는 원하는 날짜로 빠르게 이동할 수 있다. | 월 선택 또는 달력 버튼으로 계획 날짜를 선택할 수 있다. |
| TD-11 | P0 | 사용자는 로그인 없이 데이터를 유지할 수 있다. | 동일 기기/브라우저에서 다시 열어도 목록이 유지된다. |
| TD-12 | P0 | 사용자는 홈 화면에서 앱처럼 실행할 수 있다. | PWA 설치 후 standalone 화면으로 열린다. |
| TD-13 | P1 | 지정 날짜를 지나 앱을 열어도 계획을 놓치지 않는다. | 지난 미처리 예약/리스트가 오늘 목록에 표시된다. |
| TD-14 | P1 | 사용자는 인터넷 없이 기존 목록을 확인할 수 있다. | 한 번 접속한 뒤 오프라인에서 저장 데이터를 볼 수 있다. |

## 3. 개발 마일스톤

| 단계 | 범위 | 확인 결과 |
| --- | --- | --- |
| M1. 화면 프로토타입 | 오늘/데일리, 계획, 목적별 리스트 화면 | 모바일 흐름 검토 가능 |
