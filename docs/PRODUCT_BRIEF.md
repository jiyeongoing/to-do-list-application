# Swipe Todo - Product Brief

## 1. 서비스 정의

`Swipe Todo`는 오늘 할 일과 주간 계획, 반복 루틴을 분리하여 관리하고,
다음 주의 일을 스와이프로 미리 준비할 수 있는 iPhone 중심 PWA 투두리스트입니다.

## 2. 기획 배경

### 문제

- 오늘 당장 해야 할 일과 나중에 할 일이 섞이면 우선순위가 흐려집니다.
- 반복되는 루틴을 매일 다시 입력하는 것은 번거롭습니다.
- 다음 주 계획을 미리 적고 싶지만, 현재 집중해야 할 목록을 복잡하게 만들고 싶지 않습니다.

### 해결 방향

- 첫 진입 화면은 당장 필요한 `오늘 / 이번 주 / 데일리`에 집중합니다.
- 미래 계획인 `다음 주`는 스와이프 후 보이는 보조 영역으로 배치합니다.
- 반복 및 주간 전환을 자동화하여 사용자의 입력 부담을 줄입니다.

## 3. 목표 사용자

### Primary Persona

| 항목 | 내용 |
| --- | --- |
| 사용자 | iPhone을 사용하며 매일 개인 할 일을 관리하는 사람 |
| 상황 | 출근 전, 이동 중, 자기 전 짧게 할 일을 확인하거나 기록 |
| 불편 | 복잡한 캘린더보다 간단한 실행 목록이 필요함 |
| 기대 | 앱을 빠르게 열고 오늘 해야 할 일만 바로 확인 |

## 4. 핵심 가치

> 미래 계획은 준비하되, 오늘 화면은 가볍게 유지한다.

| 가치 | 제품 기능 |
| --- | --- |
| 즉시 실행 | 오늘 목록을 초기 화면으로 제공 |
| 부담 없는 계획 | 다음 주 목록은 스와이프로 접근 |
| 반복 입력 감소 | 데일리 항목을 오늘 목록에 자동 생성 |
| 자연스러운 전환 | 다음 주가 되면 이번 주 목록으로 자동 이동 |

## 5. 정보 구조

| 분류 | 의미 | 주요 행동 |
| --- | --- | --- |
| 오늘 | 오늘 실행할 목록 | 추가, 완료, 삭제 |
| 이번 주 | 이번 주 중 처리할 목록 | 추가, 오늘로 이동, 삭제 |
| 데일리 | 매일 반복할 목록 | 추가, 활성화/비활성화, 삭제 |
| 다음 주 | 다음 주를 위해 준비한 목록 | 추가, 삭제, 주간 전환 시 자동 이동 |

## 6. 핵심 정책

- 한 주의 기준은 월요일 00:00부터 일요일 23:59까지입니다.
- `다음 주` 목록은 새로운 주에 처음 앱을 열었을 때 `이번 주`로 이동합니다.
- `데일리` 활성 항목은 날짜별로 한 번만 `오늘` 목록에 생성됩니다.
- 데이터는 첫 버전에서 해당 브라우저 기기 내부에만 저장합니다.
- 브라우저 데이터 삭제 또는 기기 변경 시 복구되지 않는 점을 사용자에게 안내합니다.

## 7. MVP 성공 기준

| 확인할 가정 | 확인 방식 |
| --- | --- |
| 목록 분류가 사용자의 계획 방식에 자연스러운가 | 일주일 직접 사용 후 불필요하거나 부족한 분류 기록 |
| 다음 주 스와이프 접근이 편리한가 | 다음 주 작성 시 탐색 불편 여부 확인 |
| 로그인 없는 로컬 저장으로 첫 사용에 충분한가 | 동기화 필요가 실제로 발생하는지 확인 |

## 8. MVP 제외 범위

- 회원가입, 로그인, 서버 동기화
- 푸시 알림, 마감 시간
- 캘린더 뷰, 통계
- 공유 및 협업
- 실제 iPhone 위젯

## 9. 기술 제안

| 영역 | 제안 | 이유 |
| --- | --- | --- |
| Web | Spring Boot + Thymeleaf | Java 기반으로 빠르게 페이지 구성 |
| Storage | IndexedDB | 로그인 없이 오프라인 데이터 저장 |
| App-like UX | PWA manifest + service worker | 홈 화면 설치와 오프라인 재실행 |

첫 버전에서 Spring Boot는 화면을 제공하는 역할을 맡고, 할 일 데이터는
브라우저 내부에 저장합니다. 추후 로그인을 추가할 때 서버 API와 DB로 확장합니다.
