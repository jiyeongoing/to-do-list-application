# Swipe Todo - Product Brief

## 1. 서비스 정의

`Swipe Todo`는 오늘 할 일과 반복 루틴을 빠르게 확인하고, 미래의 할 일과
목적별 체크리스트를 날짜별로 저장해 해당 날짜에 보여주는 iPhone 중심 PWA 투두리스트입니다.

## 2. 기획 배경

### 문제

- 반복되는 루틴을 매일 다시 입력하는 것은 번거롭습니다.
- 미래에 해야 할 일을 미리 기록하고 싶지만, 오늘 화면은 복잡하지 않아야 합니다.
- 계획한 날짜가 되었을 때 다시 옮겨 적는 수고 없이 바로 실행 목록에서 보고 싶습니다.
- 장보기나 여행 준비처럼 여러 항목을 묶어 그 목록만 따로 확인하고 싶습니다.

### 해결 방향

- 첫 진입 화면은 `오늘 / 데일리`만 제공해 단순하게 유지합니다.
- 오늘 화면에서 오른쪽으로 넘긴 계획 화면에 미래 할 일을 미리 저장합니다.
- 계획 화면에서 `장보기`, `여행 준비` 같은 목적별 리스트를 만듭니다.
- 데일리와 날짜별 예약 항목을 각 날짜의 오늘 목록에 자동 생성합니다.
- 새 데일리 루틴은 기본 미사용으로 추가해 사용자가 필요한 루틴만 켭니다.

## 3. 목표 사용자

| 항목 | 내용 |
| --- | --- |
| 사용자 | iPhone을 사용하며 매일 개인 할 일을 관리하는 사람 |
| 상황 | 이동 중 또는 자기 전 오늘 할 일을 확인하고 미래 일을 기록 |
| 불편 | 복잡한 캘린더보다 바로 체크할 목록이 필요함 |
| 기대 | 앱을 열면 오늘 해야 할 일이 자동으로 모여 있음 |

## 4. 핵심 가치

> 미리 적어두고, 오늘은 실행만 한다.

| 가치 | 제품 기능 |
| --- | --- |
| 즉시 실행 | 오늘 목록을 초기 화면으로 제공 |
| 반복 입력 감소 | 데일리 항목을 오늘 목록에 자동 생성 |
| 미래 계획 정리 | 계획 화면에서 미리 입력 |
| 자동 준비 | 지정 날짜의 항목을 오늘 목록에 자동 반영 |
| 상황별 집중 | 목적별 리스트를 열어 필요한 체크 항목만 표시 |

## 5. 정보 구조

| 분류 | 의미 | 주요 행동 |
| --- | --- | --- |
| 오늘 | 오늘 실행할 목록 | 추가, 완료, 삭제 |
| 데일리 | 매일 반복할 원본 루틴 | 추가, 활성화/비활성화, 삭제 |
| 계획 | 날짜를 지정해 미리 준비할 목록 | 월/날짜 선택, 추가, 삭제 |
| 목적별 리스트 | 날짜에 연결된 별도 체크리스트 | 리스트 생성, 항목 체크, 삭제 |

## 6. 핵심 정책

- 사용자가 선택한 날짜의 예약 항목은 해당 날짜 첫 실행 때 `오늘` 목록으로 이동합니다.
- 목적별 리스트는 지정 날짜가 되면 오늘 목록에 리스트 카드로 표시하고, 상세 화면에서만 하위 항목을 체크합니다.
- 앱을 지정 날짜에 열지 못한 경우, 지난 예약 항목은 원래 날짜의 계획 기록으로 확인합니다.
- 활성 데일리 항목은 날짜별로 한 번만 `오늘` 목록에 미완료 상태로 생성됩니다.
- 날짜가 바뀌면 오늘 화면은 새 날짜 항목만 보여줍니다.
- 계획하기로 처음 들어가면 오늘 날짜가 자동 선택됩니다.
- 오늘 화면에서 오른쪽 스와이프하면 계획으로, 계획 화면에서 왼쪽 스와이프하면 오늘로 이동합니다.
- 데이터는 첫 버전에서 해당 브라우저 기기 내부에만 저장합니다.

## 7. MVP 성공 기준

| 확인할 가정 | 확인 방식 |
| --- | --- |
| 오늘/데일리 두 탭만으로 기본 사용이 충분한가 | 일주일 사용 후 탐색 불편 기록 |
| 계획 화면이 미래 할 일 기록에 편리한가 | 계획 입력 및 자동 반영 누락 여부 확인 |
| 목적별 리스트가 상황성 할 일 관리에 유용한가 | 장보기/여행 준비 목록 사용 후 분리 필요성 확인 |
| 로그인 없는 로컬 저장으로 충분한가 | 동기화 필요가 실제로 발생하는지 확인 |

## 8. 계정/동기화 확장 결정

초기 사용성은 로그인 없이 유지하되, 현재 프로토타입에는 선택형 계정 저장 흐름을
추가했습니다. 핵심 방향은 `게스트 사용 가능 + 로그인 시 계정 저장/동기화`입니다.

| 항목 | 결정 |
| --- | --- |
| 기본 사용 | 로그인 없이 계속 사용 가능 |
| 1차 로그인 | Google 로그인 프로토타입 우선 |
| 2차 로그인 | Apple 로그인 검토 |
| 로컬 데이터 | 로그인 후 사용자가 선택하면 계정으로 가져오기 |
| 저장 범위 | 오늘, 데일리, 계획, 목적별 리스트 전체 |
| 동기화 안내 | `이 기기에 저장됨`, `계정에 저장됨` 상태 표시 |

이 결정은 현재 앱의 빠른 진입 장점을 유지하면서, 여러 기기에서 이어 쓰고 싶은
사용자에게만 계정 저장을 제공하기 위한 것입니다.

## 9. MVP 제외 범위

- 푸시 알림, 마감 시간
- 캘린더 월간 보기와 통계
- 공유 및 협업
- 실제 iPhone 위젯

## 10. 기술 제안

| 영역 | 제안 | 이유 |
| --- | --- | --- |
| Web | Spring Boot + Thymeleaf | Java 기반으로 빠르게 페이지 구성 |
| Storage | LocalStorage, 이후 IndexedDB | 로그인 없이 오프라인 데이터 저장 |
| Auth | OAuth2 Login | 비밀번호 보관 부담을 줄이고 연동 로그인 제공 |
| Server Storage | PostgreSQL 또는 H2 | 계정별 투두 데이터 저장 |
| App-like UX | PWA manifest + service worker | 홈 화면 설치와 오프라인 재실행 |

현재 프로토타입은 정적 PWA와 LocalStorage로 먼저 사용성을 검증합니다.
추후 Spring Boot를 도입하면 화면 제공과 API 설계를 분리하고, 로그인한 사용자에게는
서버 저장을 제공하며 게스트 사용자는 로컬 저장을 유지합니다.
